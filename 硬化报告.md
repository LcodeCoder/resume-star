### 修复清单（本次已改）

| 级别 | 问题 | 位置 | 修复 |
|------|------|------|------|
| 🔴 严重 | **简历接口鉴权形同虚设**：拦截器写 `/resume/**`，控制器实际是 `/resumes`，模式不匹配→所有简历接口免登录可访问 | `WebMvcConfig` | 路径改 `/resumes/**`，并排除公开分享 `/resumes/share/*` |
| 🔴 严重 | **水平越权（IDOR）**：`userId` 由 `@RequestParam`/请求体传入，传 `?userId=N` 即可读写他人简历 | `ResumeController` 等 | 新增 `@CurrentUserId`，userId 统一取自 **Session**，客户端无法伪造 |

### 行为变化（需注意）

简历、面试等接口现在**必须登录**（修复前因路径 bug 实际不需要）。前端是 Session 登录、且会带 Cookie，登录后正常；**未登录的「演示模式」直接访问这些接口会返回 401** —— 这是正确的安全姿态，但若你依赖免登录演示，请确认。

此外，**社区投稿/点赞、模板收藏现在也要求登录**（未登录返回「请先登录」）；社区案例/文章/模板的**浏览**仍公开。前端调用里残留的 `userId` 入参已被后端忽略（改用会话），无需立即改但建议后续清理。

**前端体验调整（提交 `9d7ca3c`）**：后端 API 仍要求登录（安全不变），但**简历编辑器、模拟面试两个页面已放开为可匿名浏览**——进入页面只加载公开数据、以空白态起步；**保存 / 导出 / AI / 复制 / 删除 / 分享 / 开始面试 / 查看历史**等需账号的操作在执行前用 `requireLogin()` 校验，未登录弹「请先登录」并拦截。即「页面可看，操作要登录」。

## ④ 编辑器 v-if 链健壮性梳理

**起因**：本次最初修复的「图标文字重复渲染」就是一条类型分发链被中途插入的 `v-if`（字数统计 span）截断，导致兜底 `v-else` 错配、内容渲染两遍。

**梳理范围与结论**：

| 组件 | 链类型 | 结论 |
|------|--------|------|
| `DragResumeCanvas` | 组件类型分发（13 分支 + 编辑态 + 兜底） | 唯一出问题处，**已修**：字数统计 span 移到 `v-else` 之后、独立 `v-if`，链恢复连续；并加注释防回归 |
| `TemplatePreview` | 同类型分发 | 干净，单链以 `v-else` 兜底，无中断 |
| `ResumeVisual` | 图表类型分发（radar→…→statcard） | 干净，同级连续；内部 `v-if` 均嵌套在各分支内、作用域正确；无 `v-else` 无错配 |
| `InterviewView` | 面试阶段分发（intro→running→loading→report） | 干净，顶层连续；各 `v-else` 均嵌套在分支内、与相邻 `v-if` 正确配对 |
| `EditorView`/`ProfileView` 等 | 布尔开关（X / 非X） | 非分发链，逐一核对配对正确 |

**全局佐证**：`vite build` 无任何「`v-else`/`v-else-if` 缺少相邻 `v-if`」告警 → 不存在被普通元素截断的链。

**沉淀的规范**：类型/状态分发链里，**不要把独立用途的 `v-if` 元素插在最后一个 `v-else-if` 与兜底 `v-else` 之间**；这类浮层（字数、徽标等）应放在 `v-else` 之后并自带独立 `v-if`。该约定已以注释写入 `DragResumeCanvas`。

## 设计与模板优化建议（UI/UX）

> 基于当前实际样式盘点 + UI/UX 设计库（ui-ux-pro-max）给出，按"地基→页面→模板→优先级"组织。当前视觉是 **Apple/iOS 风**（系统字体 + 主色 `#5b5bd6` 紫 + 大量浅灰），底子不错，建议是**收敛与精修**，不是推倒重来。

### 1. 设计 token 地基（最该先做，成本低收益高）

盘点发现的"散"：
- **圆角 10 种**（4/6/7/10/11/12/16/18/20/24px）—— 缺统一阶梯。
- **box-shadow 47 处**—— 多为一次性手写值，没有 elevation 体系。
- **颜色多为裸 hex**（`#1d1d1f`×52、`#5b5bd6`×40、`#6e6e73`×33…）—— 改主题要全局替换。

建议：
- **圆角收敛**成一套：`6 / 8 / 12 / 16`（特大卡片 20），删掉 7/11/18 等一次性值。
- **阴影分 3 级**：卡片(sm) / 浮层(md) / 模态(lg)，统一替换 47 处散值。
- **语义色 token**：`--color-primary / text / text-muted / border / surface / success / warning / danger`，组件只引用 token、不再写裸 hex（改主题/做暗色只需改一处）。
- **字号阶梯** `12/14/16/20/24/32` + 行高（正文 1.5–1.7、标题 1.2–1.3）。
- **字体（可选）**：拉丁文用 `Plus Jakarta Sans`（SaaS 友好），中文用 PingFang/思源黑体，`font-display: swap`；保留现有系统字体也可，关键是统一一处定义。
- **暗色模式**：应用外壳（导航/侧栏/卡片/工具栏）支持暗色，简历纸张画布保持白，token 化后增量很小。

### 2. 可访问性 & 交互（直接影响"专业感"）

- **对比度**：浅灰文字 `#9e9ea4` 在白底约 2.6:1，低于 WCAG 4.5:1 → 次要文字调深到 `#6e6e73` 一档或更深；价格/数字用等宽数字避免跳动。
- **焦点态**：键盘 Tab 给可见 focus ring（2px），不要为了好看去掉。
- **动效**：统一 150–300ms、ease-out 进/ease-in 出；尊重 `prefers-reduced-motion`。
- **可点元素**：`cursor: pointer` + 清晰的 hover/active/disabled 三态；按钮异步时 loading 禁用。
- **图标**：统一用一套 SVG 图标（描边粗细一致），避免 emoji 当结构图标。

### 3. 各页面设计

- **首页 Home**：聚焦**单一主 CTA**（"免费开始制作"），次级按钮弱化；加**交互式 demo / 模板走马灯 / before-after 对比**与**社会证明**（已生成简历数、合作院校/公司 logo 墙）提升转化。
- **编辑器 Editor**（核心）：加**对齐参考线 + 吸附**、**键盘快捷键提示浮层**、**组件搜索/分组**（现工具栏偏密集）、更友好的**空画布引导**；窄屏给"建议桌面端编辑"提示（编辑器本质是桌面体验）。
- **模板库 Templates**：增加**风格标签筛选**（单栏/双栏/带照片/ATS/深色）、**骨架屏**、**hover 实时预览**、卡片上明确的"套用"CTA。
- **个人中心 Profile**：额度用**环形进度图**可视化；操作记录做**时间线**；各列表给**空状态**引导（"还没有简历，去创建"）。
- **数据可视化**（面试报告/能力雷达）：补**图例、tooltip、空/加载/错误态**，用**色盲友好**配色（别只靠红绿），动画respect reduced-motion。

### 4. 可新增的简历模板

现有 4 类约 9 套（互联网技术 / 产品运营 / 应届校园 / 设计创意）。建议沿两个维度扩充：

**新增行业分类**：商务管理、金融法律、医疗教育、学术科研、运营市场、海外英文。

**新增"结构型"模板（跨行业复用，价值最高）**：
| 模板 | 卖点 | 适合 |
|------|------|------|
| **ATS 友好单栏** | 无图标/无分栏，便于招聘系统机器解析 | 求职刚需，几乎所有人 |
| **经典双栏** | 左信息栏 + 右主体，信息密度高 | 通用、有经验者 |
| **时间线型** | 左侧时间轴串联经历 | 经历连贯的求职者 |
| **高管/资深** | 强调成果与履历，留白克制 | 管理/资深岗 |
| **作品集型** | 强视觉 + 项目卡片 | 设计/前端/创意 |
| **学术 CV** | 多页，论文/项目/获奖列表 | 科研/读研/申博 |

| **英文/海外** | 拉丁字体、无照片、A4 ATS | 外企/海外申请 |

**模板能力增强（关键）**：让每套模板支持**配色切换 / 字体切换 / 单双栏切换**——一套结构衍生多种风格，用更少维护成本覆盖更多场景。

### 5. 落地优先级

- **P0**（低成本高收益）：token 收敛（圆角/阴影/语义色）、对比度修正、focus 态、统一动效时长。✅ **已实施**（提交 `c17be0b`/`5e81f7c`）：`:root` 建立语义色/圆角(7级+pill)/中性阴影(3级)/动效/焦点 token；圆角 64 处、中性阴影 16 处迁移到 token（品牌发光/动画/inset 阴影保留）；浅灰文字 `#9e9ea4`(2.6:1)→`#6e6e73`(4.6:1, WCAG AA) 20 处；新增 `:focus-visible` 焦点环与 `prefers-reduced-motion` 守卫。

``` 
 import okhttp3.*;
import com.fasterxml.jackson.databind.ObjectMapper;
import java.io.IOException;
import java.util.HashMap;
import java.util.Map;
import java.util.concurrent.TimeUnit;

/**
 * Java 单文件实现GET/POST(JSON) API请求
 * 基于OkHttp和Jackson，复制后直接使用（需添加Maven/Gradle依赖）
 */
public class ApiRequest {
    // 全局OkHttpClient实例（复用连接池）
    private static final OkHttpClient OK_HTTP_CLIENT = new OkHttpClient.Builder()
            .connectTimeout(10, TimeUnit.SECONDS)
            .readTimeout(10, TimeUnit.SECONDS)
            .writeTimeout(10, TimeUnit.SECONDS)
            .build();

    // Jackson JSON解析工具
    private static final ObjectMapper OBJECT_MAPPER = new ObjectMapper();

    /**
     * GET请求方法
     * @param url 接口地址
     * @param params URL参数键值对
     * @return 接口响应字符串
     * @throws IOException 网络异常
     */
    public static String doGet(String url, Map params) throws IOException {
        HttpUrl.Builder urlBuilder = HttpUrl.parse(url).newBuilder();
        if (params != null && !params.isEmpty()) {
            for (Map.Entry entry : params.entrySet()) {
                urlBuilder.addQueryParameter(entry.getKey(), entry.getValue());
            }
        }
        Request request = new Request.Builder().url(urlBuilder.build()).get().build();
        try (Response response = OK_HTTP_CLIENT.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                throw new IOException("GET请求失败：" + response.code() + " " + response.message());
            }
            return response.body() != null ? response.body().string() : "";
        }
    }

    /**
     * POST(JSON)请求方法
     * @param url 接口地址
     * @param jsonStr JSON请求体字符串
     * @return 接口响应字符串
     * @throws IOException 网络异常
     */
    public static String doPostJson(String url, String jsonStr) throws IOException {
        MediaType mediaType = MediaType.parse("application/json; charset=utf-8");
        RequestBody requestBody = RequestBody.create(mediaType, jsonStr);
        Request request = new Request.Builder().url(url).post(requestBody).build();
        try (Response response = OK_HTTP_CLIENT.newCall(request).execute()) {
            if (!response.isSuccessful()) {
                throw new IOException("POST请求失败：" + response.code() + " " + response.message());
            }
            return response.body() != null ? response.body().string() : "";
        }
    }

    /**
     * 格式化JSON字符串（便于阅读）
     * @param jsonStr 原始JSON字符串
     * @return 格式化后的JSON
     * @throws IOException 解析异常
     */
    public static String formatJson(String jsonStr) throws IOException {
        if (jsonStr == null || jsonStr.isEmpty()) {
            return "";
        }
        Object jsonObj = OBJECT_MAPPER.readValue(jsonStr, Object.class);
        return OBJECT_MAPPER.writerWithDefaultPrettyPrinter().writeValueAsString(jsonObj);
    }

    // 主方法：测试GET/POST请求
    public static void main(String[] args) {
        try {
            // ------------------- 测试GET请求 -------------------
            String getUrl = "https://api.example.com/user/list"; // 替换为实际接口地址
            Map getParams = new HashMap<>();
            getParams.put("page", "1");
            getParams.put("size", "10");
            getParams.put("keyword", "test");
            String getResponse = doGet(getUrl, getParams);
            System.out.println("=== GET请求响应结果 ===");
            System.out.println(formatJson(getResponse));

            System.out.println("------------------------");

            // ------------------- 测试POST请求 -------------------
            String postUrl = "https://api.example.com/user/login"; // 替换为实际接口地址
            Map postData = new HashMap<>();
            postData.put("username", "admin");
            postData.put("password", "123456");
            String jsonStr = OBJECT_MAPPER.writeValueAsString(postData);
            String postResponse = doPostJson(postUrl, jsonStr);
            System.out.println("=== POST请求响应结果 ===");
            System.out.println(formatJson(postResponse));

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 沉浸式语音面试 — 已实现功能（验收清单）

> 实现人：Claude。涉及文件：后端 `entity/SystemConfig`、`service/impl/SystemConfigServiceImpl`、`controller/InterviewController`、`service/InterviewService(+Impl)`、`entity/InterviewAnswerRequest`、`ai/CloudTtsClient`、`config/WebMvcConfig`、`config/RateLimitInterceptor`；前端 `views/InterviewView.vue`、`composables/useSpeech.js`、`components/common/LineIcon.vue`、`views/admin/AdminInterviewTab.vue`。

- **沉浸式入口与扣费**：标准面试外新增「沉浸式语音面试」入口；单场消耗额度（默认 2）、时长（默认 30 分钟）、是否开启，均由管理端配置；首题前按消耗核验余额。
- **语音提问（TTS）**：AI 问题用语音念出；**云端神经网络音色**（hewoyi，晓晓/云希等 6 个 zh-CN 音色，密钥仅后端持有）与**浏览器本地音色**可切换、可试听；云端不可用自动回退浏览器并提示。
  - 云端采用「服务端解析音频地址 → 浏览器直连 CDN 播放」两段式，规避部分服务器到 CDN 链路不稳的问题；JVM 加 `-Djava.net.preferIPv4Stack=true`。
- **语音作答（STT）**：浏览器 Web Speech API 实时转写；不支持的浏览器自动降级为打字。
- **全程免手操（默认开）**：面试官说完自动开麦，用户停顿 ~2.6s 自动停止并提交、自动进入下一题；可一键切回手动。
- **临场感**：面试官头像说话时脉动光环、录音时实时音量波形；TTS 串行队列，杜绝多声叠加。
- **表达力分析**：报告含语速（字/分）、平均作答字数/时长、口头禅次数与改进建议。
- **结束语音播报**：报告生成后语音播报综合评价与鼓励语，可手动重播。
- **移动端**：全站响应式适配——顶部导航在窄屏收为汉堡抽屉、隐藏冗长额度文案；首页/模板库/社区/个人中心/会员/登录等栅格自动单列；编辑器给「建议桌面端」提示并让工具栏换行；后台标签栏与表格横向滚动；弹窗占满窄屏；面试全流程（对话/工具条/报告/波形）单列适配。
