## ② 安全审计与修复

### 修复清单（本次已改）

| 级别 | 问题 | 位置 | 修复 |
|------|------|------|------|
| 🔴 严重 | **简历接口鉴权形同虚设**：拦截器写 `/resume/**`，控制器实际是 `/resumes`，模式不匹配→所有简历接口免登录可访问 | `WebMvcConfig` | 路径改 `/resumes/**`，并排除公开分享 `/resumes/share/*` |
| 🔴 严重 | **水平越权（IDOR）**：`userId` 由 `@RequestParam`/请求体传入，传 `?userId=N` 即可读写他人简历 | `ResumeController` 等 | 新增 `@CurrentUserId`，userId 统一取自 **Session**，客户端无法伪造 |
| 🟠 高 | `versions`/`restore`/`share`/`getShare` **完全无归属校验**，凭 resumeId 即可操作任意简历 | `ResumeController` | 增加 `notOwned()` 归属校验 |
| 🟠 高 | 面试记录、会员兑换/流水 **免登录 + IDOR** | `InterviewController`、`MemberController` | 纳入拦截 + `@CurrentUserId`；提交以登录态覆盖请求体 userId |
| 🟡 中 | **上传后缀取自用户文件名**：`image/png` 内容配 `evil.html` 后缀→落地为可被当 HTML 解析的文件（存储型 XSS） | `FileUploadServiceImpl` | 扩展名改由**已校验的 contentType** 推导；上传纳入鉴权 |
| 🔴 严重 | **社区审核/删除接口无鉴权**：`approve`、`/community/admin/*`、删除对匿名开放，任何人可审核/删除任意内容 | `CommunityController` | `approve`/后台列表限管理员；删除限**管理员或作者本人**；按简历删投稿仅限本人 |
| 🟠 高 | **社区投稿/点赞、模板收藏 IDOR**：`userId` 由客户端传入 | `Community`/`TemplateController` | 改 `@CurrentUserId`（会话注入），变更类操作要求登录 |

### 行为变化（需注意）

简历、面试等接口现在**必须登录**（修复前因路径 bug 实际不需要）。前端是 Session 登录、且会带 Cookie，登录后正常；**未登录的「演示模式」直接访问这些接口会返回 401** —— 这是正确的安全姿态，但若你依赖免登录演示，请确认。

此外，**社区投稿/点赞、模板收藏现在也要求登录**（未登录返回「请先登录」）；社区案例/文章/模板的**浏览**仍公开。前端调用里残留的 `userId` 入参已被后端忽略（改用会话），无需立即改但建议后续清理。

**前端体验调整（提交 `9d7ca3c`）**：后端 API 仍要求登录（安全不变），但**简历编辑器、模拟面试两个页面已放开为可匿名浏览**——进入页面只加载公开数据、以空白态起步；**保存 / 导出 / AI / 复制 / 删除 / 分享 / 开始面试 / 查看历史**等需账号的操作在执行前用 `requireLogin()` 校验，未登录弹「请先登录」并拦截。即「页面可看，操作要登录」。

## ④ 编辑器 v-if 链健壮性梳理

**起因**：本次最初修复的「图标文字重复渲染」就是一条类型分发链被中途插入的 `v-if`（字数统计 span）截断，导致兜底 `v-else` 错配、内容渲染两遍。

**梳理范围与结论**：

| 组件 | 链类型 | 结论 |
|------|--------|------|
| `DragResumeCanvas` | 组件类型分发（13 分支 + 编辑态 + 兜底） | 唯一出问题处，**已修**：字数统计 span 移到 `v-else` 之后、独立 `v-if`，链恢复连续；并加注释防回归 |
| `TemplatePreview` | 同类型分发 | 干净，单链以 `v-else` 兜底，无中断 |
| `ResumeVisual` | 图表类型分发（radar→…→statcard） | 干净，同级连续；内部 `v-if` 均嵌套在各分支内、作用域正确；无 `v-else` 无错配 |
| `InterviewView` | 面试阶段分发（intro→running→loading→report） | 干净，顶层连续；各 `v-else` 均嵌套在分支内、与相邻 `v-if` 正确配对 |
| `EditorView`/`ProfileView` 等 | 布尔开关（X / 非X） | 非分发链，逐一核对配对正确 |

**全局佐证**：`vite build` 无任何「`v-else`/`v-else-if` 缺少相邻 `v-if`」告警 → 不存在被普通元素截断的链。

**沉淀的规范**：类型/状态分发链里，**不要把独立用途的 `v-if` 元素插在最后一个 `v-else-if` 与兜底 `v-else` 之间**；这类浮层（字数、徽标等）应放在 `v-else` 之后并自带独立 `v-if`。该约定已以注释写入 `DragResumeCanvas`。

## 设计与模板优化建议（UI/UX）

> 基于当前实际样式盘点 + UI/UX 设计库（ui-ux-pro-max）给出，按"地基→页面→模板→优先级"组织。当前视觉是 **Apple/iOS 风**（系统字体 + 主色 `#5b5bd6` 紫 + 大量浅灰），底子不错，建议是**收敛与精修**，不是推倒重来。

### 1. 设计 token 地基（最该先做，成本低收益高）

盘点发现的"散"：
- **圆角 10 种**（4/6/7/10/11/12/16/18/20/24px）—— 缺统一阶梯。
- **box-shadow 47 处**—— 多为一次性手写值，没有 elevation 体系。
- **颜色多为裸 hex**（`#1d1d1f`×52、`#5b5bd6`×40、`#6e6e73`×33…）—— 改主题要全局替换。

建议：
- **圆角收敛**成一套：`6 / 8 / 12 / 16`（特大卡片 20），删掉 7/11/18 等一次性值。
- **阴影分 3 级**：卡片(sm) / 浮层(md) / 模态(lg)，统一替换 47 处散值。
- **语义色 token**：`--color-primary / text / text-muted / border / surface / success / warning / danger`，组件只引用 token、不再写裸 hex（改主题/做暗色只需改一处）。
- **字号阶梯** `12/14/16/20/24/32` + 行高（正文 1.5–1.7、标题 1.2–1.3）。
- **字体（可选）**：拉丁文用 `Plus Jakarta Sans`（SaaS 友好），中文用 PingFang/思源黑体，`font-display: swap`；保留现有系统字体也可，关键是统一一处定义。
- **暗色模式**：应用外壳（导航/侧栏/卡片/工具栏）支持暗色，简历纸张画布保持白，token 化后增量很小。

### 2. 可访问性 & 交互（直接影响"专业感"）

- **对比度**：浅灰文字 `#9e9ea4` 在白底约 2.6:1，低于 WCAG 4.5:1 → 次要文字调深到 `#6e6e73` 一档或更深；价格/数字用等宽数字避免跳动。
- **焦点态**：键盘 Tab 给可见 focus ring（2px），不要为了好看去掉。
- **动效**：统一 150–300ms、ease-out 进/ease-in 出；尊重 `prefers-reduced-motion`。
- **可点元素**：`cursor: pointer` + 清晰的 hover/active/disabled 三态；按钮异步时 loading 禁用。
- **图标**：统一用一套 SVG 图标（描边粗细一致），避免 emoji 当结构图标。

### 3. 各页面设计

- **首页 Home**：聚焦**单一主 CTA**（"免费开始制作"），次级按钮弱化；加**交互式 demo / 模板走马灯 / before-after 对比**与**社会证明**（已生成简历数、合作院校/公司 logo 墙）提升转化。
- **编辑器 Editor**（核心）：加**对齐参考线 + 吸附**、**键盘快捷键提示浮层**、**组件搜索/分组**（现工具栏偏密集）、更友好的**空画布引导**；窄屏给"建议桌面端编辑"提示（编辑器本质是桌面体验）。
- **模板库 Templates**：增加**风格标签筛选**（单栏/双栏/带照片/ATS/深色）、**骨架屏**、**hover 实时预览**、卡片上明确的"套用"CTA。
- **个人中心 Profile**：额度用**环形进度图**可视化；操作记录做**时间线**；各列表给**空状态**引导（"还没有简历，去创建"）。
- **数据可视化**（面试报告/能力雷达）：补**图例、tooltip、空/加载/错误态**，用**色盲友好**配色（别只靠红绿），动画respect reduced-motion。

### 4. 可新增的简历模板

现有 4 类约 9 套（互联网技术 / 产品运营 / 应届校园 / 设计创意）。建议沿两个维度扩充：

**新增行业分类**：商务管理、金融法律、医疗教育、学术科研、运营市场、海外英文。

**新增"结构型"模板（跨行业复用，价值最高）**：
| 模板 | 卖点 | 适合 |
|------|------|------|
| **ATS 友好单栏** | 无图标/无分栏，便于招聘系统机器解析 | 求职刚需，几乎所有人 |
| **经典双栏** | 左信息栏 + 右主体，信息密度高 | 通用、有经验者 |
| **时间线型** | 左侧时间轴串联经历 | 经历连贯的求职者 |
| **高管/资深** | 强调成果与履历，留白克制 | 管理/资深岗 |
| **作品集型** | 强视觉 + 项目卡片 | 设计/前端/创意 |
| **学术 CV** | 多页，论文/项目/获奖列表 | 科研/读研/申博 |
| **深色简历** | 暗色高对比 | 设计/技术展示 |
| **英文/海外** | 拉丁字体、无照片、A4 ATS | 外企/海外申请 |

**模板能力增强（关键）**：让每套模板支持**配色切换 / 字体切换 / 单双栏切换**——一套结构衍生多种风格，用更少维护成本覆盖更多场景。

### 5. 落地优先级

- **P0**（低成本高收益）：token 收敛（圆角/阴影/语义色）、对比度修正、focus 态、统一动效时长。✅ **已实施**（提交 `c17be0b`/`5e81f7c`）：`:root` 建立语义色/圆角(7级+pill)/中性阴影(3级)/动效/焦点 token；圆角 64 处、中性阴影 16 处迁移到 token（品牌发光/动画/inset 阴影保留）；浅灰文字 `#9e9ea4`(2.6:1)→`#6e6e73`(4.6:1, WCAG AA) 20 处；新增 `:focus-visible` 焦点环与 `prefers-reduced-motion` 守卫。
- **P1**：模板库风格筛选 + 骨架屏；新增 **ATS 友好单栏** + **经典双栏** 两套（求职最刚需）。
- **P2**：暗色模式；编辑器对齐吸附 + 快捷键提示；首页交互 demo + 社会证明。
