# 履历星图前端架构

## 设计命题

产品是一座面向求职者的“轨道工作站”。用户不是在浏览仪表盘，而是在沿着简历、模板、面试和投递四条轨道推进任务。宇宙意象只承担方向、状态和品牌记忆，不替代标准表单、表格和编辑器交互。

## 信息架构

```text
应用入口
├── 登录 / 注册 / 管理员验证（独立星门页）
├── 任务总览（当前航程 + 三项核心任务）
├── 简历工坊（内容编辑 + 画布预览 + 导出）
├── 模板星库（筛选 + 列表 + 详情预览）
├── 面试舱（配置 + 对话 + 报告）
├── 社区信号（案例 + 教程 + 详情）
├── 个人档案（资料 + 简历版本 + 额度记录）
├── 会员航程（权益 + 套餐 + 订单）
├── 系统设置（显示 + 编辑 + 通知 + 隐私）
├── 控制中心（管理后台的业务标签页）
└── 未知区域（404 + 返回路径）
```

桌面使用固定轨道导航与独立内容舞台。移动端导航变为左侧抽屉，页面标题与云端状态保留在顶部。登录页脱离应用壳层，避免用户在验证前接触受保护导航。

## 四套主题

所有主题在 `src/style/index.css` 中拥有完整、独立的背景、表面、文字、边界、强调和语义色。组件不得用明暗翻转或 `filter` 模拟主题。

| 标识 | 使用环境 | 背景取向 | 强调色 |
| --- | --- | --- | --- |
| `day-standard` | 日间、正常光线 | 冷白与淡蓝灰 | 高对比钴蓝 |
| `day-eye` | 长时间阅读、偏暖环境光 | 低亮度绿灰 | 深常青绿 |
| `night-standard` | 暗室、高对比需求 | 深靛黑 | 明亮青蓝 |
| `night-eye` | 暗室、长时间编辑 | 低对比深青绿 | 柔和薄荷绿 |

正文目标对比度不低于 4.5:1，大字号不低于 3:1。占位文字使用 `--muted`，禁止使用浏览器默认浅灰。主题选择保存在 `orbit_resume_theme`。

## 组件语言

- 导航：恒星品牌 + 垂直轨道 + 行星节点；当前节点必须同时具备形状、颜色和文字状态。
- 按钮：6px 圆角，主操作使用单色强调面；不使用渐变、发光大阴影或装饰图标。
- 表单：标准标签在上，聚焦态使用 2px 强调色内描边，错误必须包含文字说明。
- 表格：表头使用第二层表面色，悬停只改变背景，不改变布局。
- 卡片：仅用于单个可复用对象、弹窗和工具面板；页面区块使用分隔线与留白，不套卡片。
- 弹窗与抽屉：保留 Element Plus 的可访问性和焦点管理，通过全局语义令牌统一外观。
- 空状态：说明当前状态与下一步操作；404 使用“失去信号”的轨道图，但保留标准返回按钮。

## 动效

路由切换 160ms，导航和控件状态 180ms，移动抽屉 220ms，统一使用 `cubic-bezier(.16,1,.3,1)`。动效只表达状态变化。`prefers-reduced-motion: reduce` 下全部转换为即时更新。

## 代码边界

- `src/utils/theme.js`：唯一主题状态与持久化入口。
- `src/components/common/ThemeSwitcher.vue`：四主题切换器。
- `src/layout/MainLayout.vue`：应用导航、会话状态、页面舞台与全局协议入口。
- `src/style/index.css`：主题令牌、Element Plus 映射、应用壳层和跨页面基础组件。
- `src/views/*`：页面业务结构；页面可定义局部布局，但颜色必须来自全局语义令牌。

新增界面禁止重新引入旧式顶部办公导航、白色卡片网格、紫色渐变按钮、玻璃拟态容器或超大圆角。任何新颜色先进入四套主题令牌，再由组件消费。
