# 新增功能说明

本次更新新增了5个功能，详细说明如下：

## 1. 修复头像组件上传图像功能

### 实现内容
- 创建了 `FileUploadServiceImpl` 实现文件上传服务
- 支持 JPG、PNG、GIF 格式的图片上传
- 文件大小限制为 5MB
- 上传的文件保存在 `uploads/avatars/` 目录
- 文件名格式：`userId_uuid.扩展名`

### API 接口
- **POST** `/api/upload/avatar`
- 参数：`file` (MultipartFile)
- 返回：上传成功后的文件URL

### 使用示例
```javascript
const formData = new FormData();
formData.append('file', file);
const response = await axios.post('/api/upload/avatar', formData);
console.log(response.data.data); // 头像URL
```

## 2. 邮箱验证注册功能

### 实现内容
- 管理员可在后台配置是否启用邮箱验证注册
- 管理员可配置 QQ 邮箱 SMTP 服务器参数
- 用户注册时需要输入邮箱并获取验证码
- 验证码6位数字，有效期5分钟
- 演示环境：验证码会打印到控制台

### 相关实体
- `EmailVerifyCode` - 邮箱验证码实体
- `SystemConfig` - 系统配置实体

### API 接口

#### 2.1 发送验证码
- **POST** `/api/user/send-code`
- 参数：`email` (String)
- 返回：发送结果

#### 2.2 获取系统配置
- **GET** `/api/user/system-config`
- 返回：系统配置（包含是否启用邮箱验证）

#### 2.3 注册（支持邮箱验证）
- **POST** `/api/user/register`
- 参数：
  ```json
  {
    "username": "用户名",
    "password": "密码",
    "nickname": "昵称",
    "email": "邮箱地址",
    "emailCode": "验证码"
  }
  ```

### 管理员配置接口

#### 2.4 获取系统配置（管理员）
- **GET** `/api/admin/system-config`
- 需要管理员权限

#### 2.5 更新系统配置（管理员）
- **PUT** `/api/admin/system-config`
- 参数：
  ```json
  {
    "emailVerifyEnabled": true,
    "emailUsername": "your-email@qq.com",
    "emailPassword": "授权码"
  }
  ```

### 配置 QQ 邮箱步骤
1. 登录 QQ 邮箱
2. 进入 设置 -> 账户 -> POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务
3. 开启 SMTP 服务
4. 生成授权码
5. 在管理后台配置：
   - 发送账号：你的 QQ 邮箱
   - 授权码：刚才生成的授权码
   - SMTP 固定使用 `smtp.qq.com:465` SSL，无需手动填写

## 3. 单IP登录限制功能

### 实现内容
- 管理员可开启/关闭单IP登录限制
- 开启后，同一账号只能在一个IP登录
- 新设备登录会自动踢出旧设备
- 记录用户登录IP和Session ID

### 相关实体
- `UserLoginSession` - 用户登录会话实体

### 工作原理
1. 用户登录时记录IP地址和Session ID
2. 如果开启了单IP限制，检查是否存在其他设备登录
3. 存在则标记旧会话为失效
4. 前端检测到会话失效后提示用户重新登录

### 管理员配置
在系统配置中设置 `singleIpEnabled` 为 `true` 即可开启。

## 4. 每日导出数量限制功能

### 实现内容
- 管理员可设置用户每天可导出的模板数量
- 设置为 0 表示不限制
- 超过限制后提示用户次日再试
- 每日0点自动重置统计

### API 接口
导出接口已集成限制检查：
- **POST** `/api/export/record`
- 超限时返回错误信息：`今日导出次数已达上限（X次），请明天再试`

### 管理员配置
在系统配置中设置 `dailyExportLimit`：
- `0` - 不限制
- `3` - 每天3次
- `10` - 每天10次
- 等等...

## 5. 草稿箱功能

### 实现内容
- 用户可以将简历保存为草稿
- 草稿箱独立展示，不与正式简历混在一起
- 支持草稿发布为正式简历
- 支持删除草稿或正式简历

### API 接口

#### 5.1 查询所有简历（包含草稿和正式）
- **GET** `/api/resumes?userId=1`
- 返回：所有简历列表

#### 5.2 查询草稿箱
- **GET** `/api/resumes/drafts?userId=1`
- 返回：仅草稿列表

#### 5.3 查询正式简历
- **GET** `/api/resumes/published?userId=1`
- 返回：仅正式简历列表

#### 5.4 保存简历（草稿或正式）
- **POST** `/api/resumes`
- 参数：
  ```json
  {
    "id": null,
    "title": "简历标题",
    "targetJob": "目标岗位",
    "draft": true,
    "components": [],
    "style": {}
  }
  ```
- `draft` 为 `true` 表示保存为草稿，`false` 表示正式简历

#### 5.5 发布草稿
- **PUT** `/api/resumes/{resumeId}/publish?userId=1`
- 将草稿转为正式简历

#### 5.6 删除简历
- **DELETE** `/api/resumes/{resumeId}?userId=1`
- 删除草稿或正式简历

### 前端实现建议
1. 在个人中心或简历列表页添加"草稿箱"标签页
2. 编辑器中添加"保存为草稿"和"发布"按钮
3. 草稿列表显示草稿标识和最后编辑时间
4. 提供"发布"和"删除"操作按钮

## 配置文件说明

在 `application.yml` 中新增配置项：

```yaml
resume:
  system:
    # 是否启用邮箱验证注册
    email-verify-enabled: false
    # 邮箱账号（环境变量）
    email-username: ${RESUME_EMAIL_USERNAME:}
    # 邮箱授权码（环境变量）
    email-password: ${RESUME_EMAIL_PASSWORD:}
    # 是否启用单IP登录限制
    single-ip-enabled: false
    # 每日导出次数限制（0表示不限制）
    daily-export-limit: 0
```

## 注意事项

1. **邮箱验证**：使用 QQ 邮箱 SMTP `smtp.qq.com:465` SSL 发送验证码，需要配置发送账号和授权码
2. **单IP登录**：使用内存存储会话信息，重启后会清空，生产环境建议使用 Redis
3. **导出限制**：统计数据存储在内存，重启后会清空，生产环境建议使用数据库
4. **草稿箱**：当前使用内存存储，生产环境需要持久化到数据库
