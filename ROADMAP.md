# 📋 产品优化与功能路线图

> 基于当前 MVP 版本的改进建议，按优先级和实现难度分类

## 🔴 高优先级（用户体验关键）

### 1. 移动端适配优化
**现状**：当前设计以桌面端为主，移动端体验欠佳  
**建议**：
- [ ] 简历编辑器在手机上支持触摸拖拽
- [ ] 管理后台表格在小屏幕下横向滚动或卡片式展示
- [ ] 响应式布局优化（特别是 768px 以下断点）

**优先级理由**：简历类产品移动端使用场景占比高（地铁、咖啡厅临时修改）

---

### 2. 简历模板系统
**现状**：模板星库 + VIP 模板 + 预览/套用已上线  
**建议**：
- [x] 3-5 套预设模板（经典、现代、极简、创意）
- [x] 模板预览与一键切换
- [x] 每个模板独立的 CSS 样式和布局规则
- [x] VIP 会员专属高级模板

**商业价值**：差异化卖点，提升会员转化率

---

### 3. 实时协作与分享
**现状**：只读分享链接 + 浏览统计已上线；密码/有效期可继续增强  
**建议**：
- [x] 生成简历只读分享链接（可设置密码和有效期）
- [x] HR 直接访问链接查看简历，无需下载
- [x] 分享统计（浏览次数、停留时长）

**场景**：投递简历时附带在线链接，HR 体验更好，也能追踪简历是否被打开

---

## 🟡 中优先级（功能完善）

### 4. AI 功能增强
**现状**：AI 仅支持文本润色  
**建议**：
- [x] **职位匹配度分析**：上传 JD，AI 分析简历与岗位的匹配度并给出优化建议
- [ ] **面试问题预测**：基于简历内容生成可能被问到的面试题
- [ ] **简历对比**：多版本简历 A/B 对比，AI 推荐更优版本
- [ ] **AI 生成封面信**：根据简历和目标公司生成求职信

**技术方案**：扩展 `AiFeatureType` 枚举，复用现有 HTTP 客户端

---

### 5. 数据可视化看板
**现状**：用户端无数据统计  
**建议**：
- [ ] 用户个人看板：
  - 简历被查看/下载次数（需实现分享追踪）
  - AI 使用次数与额度统计
  - 简历版本历史对比
- [ ] 管理员增强看板：
  - 用户留存率曲线
  - 会员续费率分析
  - AI 功能使用热力图

**实现难度**：中等，前端用 ECharts，后端新增埋点接口

---

### 6. 简历版本管理
**现状**：版本列表与回滚已上线；diff 对比仍可增强  
**建议**：
- [x] 自动保存历史快照（每次大改动或手动保存时）
- [x] 版本列表查看与回滚
- [ ] 版本对比（diff 高亮显示差异）

**技术方案**：在 `rl_resume` 表新增 `history` 字段（JSON 数组存储历史版本）

---

## 🎨 前端样式优化细节

### 编辑器体验
- [ ] **拖拽排序动画**：增加平滑过渡效果（现有拖拽较生硬）
- [ ] **快捷键支持**：Ctrl+S 保存，Ctrl+Z 撤销，Ctrl+P 预览
- [ ] **实时字数统计**：在编辑框底部显示当前字数（特别是自我介绍）
- [ ] **焦点状态优化**：当前编辑框高亮，其他区域半透明

### 视觉设计
- [ ] **深色模式**：夜间编辑护眼
- [ ] **品牌色调整**：当前蓝色偏普通，考虑更有辨识度的配色
- [ ] **加载动画**：AI 请求时增加骨架屏或进度条（现在只有 loading）
- [ ] **空状态插画**：无简历时显示引导插画而非空白页

### 交互细节
- [ ] **Tooltip 提示**：图标按钮增加悬浮说明
- [ ] **确认弹窗优化**：删除操作前二次确认，防止误删
- [ ] **表单校验提示**：实时校验而非提交时才报错
- [ ] **操作反馈**：保存成功、删除成功用 Toast 而非 Alert

---

## 🔧 技术债务与重构

### 代码质量
- [ ] **前端组件拆分**：部分组件超过 500 行，拆分为子组件
- [ ] **TypeScript 迁移**：前端目前是纯 JS，迁移到 TS 提升可维护性
- [ ] **单元测试**：核心业务逻辑添加测试（至少覆盖 AI 客户端、权限校验）
- [ ] **错误边界**：前端增加全局错误捕获，避免白屏

### 性能优化
- [ ] **前端懒加载**：路由级别代码分割
- [ ] **图片优化**：头像、Logo 使用 WebP 格式
- [ ] **接口缓存**：用户信息、配置项等接口增加前端缓存
- [ ] **SQL 索引**：高频查询字段添加索引（如 `username`、`email`）

### 安全加固
- [ ] **XSS 防护**：用户输入内容统一转义
- [ ] **CSRF Token**：敏感操作（修改密码、删除）增加 CSRF 校验
- [ ] **限流策略**：AI 请求、注册接口增加单 IP 限流
- [ ] **敏感信息脱敏**：日志中完全隐藏 API Key（目前还会打印前 10 位）
